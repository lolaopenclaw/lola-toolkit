# OpenClaw Token Mismatch Fix (2026-02-28)

## Problema
Después de cada `openclaw gateway restart`, el CLI fallaba con:
```
gateway connect failed: Error: unauthorized: gateway token mismatch
```

## Causa
El gateway genera un token nuevo en cada arranque y lo guarda en `~/.openclaw/openclaw.json`. El CLI no lo estaba usando correctamente.

Las env vars necesarias:
- `OPENCLAW_GATEWAY_TOKEN`
- `OPENCLAW_GATEWAY_REMOTE_TOKEN`

Sin estas, el CLI falla incluso aunque el token esté en config.

## Solución Implementada
Crear un wrapper en `~/.local/bin/openclaw` que auto-inyecta el token:

```bash
#!/bin/bash
TOKEN=$(cat ~/.openclaw/openclaw.json | jq -r '.gateway.auth.token // ""')
if [ -n "$TOKEN" ]; then
  export OPENCLAW_GATEWAY_REMOTE_TOKEN="$TOKEN"
  export OPENCLAW_GATEWAY_TOKEN="$TOKEN"
fi
exec /home/mleon/.npm-global/bin/openclaw "$@"
```

Y agregar a `~/.bashrc`:
```bash
export PATH="$HOME/.local/bin:$PATH"
```

✅ Todos los comandos openclaw usan el token correcto automáticamente
✅ Sin necesidad de resetear gateway entre comandos
