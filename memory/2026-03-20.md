## 🧠 Memory Search Reindex Cron — Error Fix

**Problem:** Cron `53577b95` (4:30 AM daily) showed `error` status.
- The reindex itself **succeeded** (`openclaw memory index --force` ran fine, 221 files indexed)
- Error was **delivery failure**: `delivery.channel: "last"` but isolated session has no previous channel
- With both Telegram + Discord configured, OpenClaw couldn't pick one automatically

**Fix applied:**
- `openclaw cron edit 53577b95 --channel telegram --best-effort-deliver`
- Now delivers to Telegram explicitly
- `bestEffort: true` means delivery glitches won't mark the job as failed
