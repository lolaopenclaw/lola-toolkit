## Sesión nocturna (~20:34-21:10) — Duplicate gateway fix

### 🔴 Root cause de conflictos Telegram (DESCUBIERTO)
- **Dos servicios systemd de openclaw-gateway coexistían:**
  1. **Sistema** (`/etc/systemd/system/openclaw-gateway.service`) → puerto 18789
  2. **Usuario** (`~/.config/systemd/user/openclaw-gateway.service`) → puerto 18790
- Ambos enabled y arrancaban al boot → 2 gateways compitiendo por Telegram getUpdates
- Síntomas: 409 Conflict, session locks, Chrome relay EADDRINUSE, Bonjour name conflicts

### Diagnóstico
- `ps aux` mostraba 2 procesos openclaw-gateway siempre
- `pstree` reveló: uno hijo de openclaw (legítimo), otro PPID=1 (huérfano)
- `sudo systemctl status openclaw-gateway` reveló el servicio global
- El servicio global era de una instalación anterior, nunca detectado

### Fix aplicado
```bash
sudo systemctl stop openclaw-gateway
sudo systemctl disable openclaw-gateway
```
- Post-reboot: 1 solo gateway, 0 conflictos ✅
- Issue GitHub #33103 (locked, no se pudo comentar)
